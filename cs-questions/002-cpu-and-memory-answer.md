# CS 기초 질문 #2: CPU와 메모리 - 정답

**날짜**: 2025-11-03

## 당신의 답변

> CPU는 중앙 처리 장치로 모든 프로레스를 관리해
> RAM은 임시 저장 공간으로 프로그램들의 메모리를 관리하지
> 예를 들어 여러 프로그램이 실행되면 각각 프로세스들의 메모리 관리를 CPU가 담당해.
> 해당 프로세스들의 메모리는 RAM에 저장되어 관리돼

## 피드백

기본 개념은 잘 이해하고 계십니다! 다만 몇 가지 보완할 점이 있습니다.

### 맞은 부분 ✅
- CPU는 중앙 처리 장치
- RAM은 임시 저장 공간
- 프로세스들의 데이터가 RAM에 저장됨

### 수정/보완이 필요한 부분 ⚠️

1. **CPU의 역할**: "프로세스 관리"보다는 **"연산 및 명령어 실행"**이 주된 역할
   - CPU는 계산하고, 명령을 실행하는 "두뇌"
   - 프로세스 관리는 운영체제(OS)가 담당

2. **RAM의 역할**: "메모리를 관리"한다기보다 **"데이터를 저장"**하는 공간
   - RAM 자체는 저장 공간일 뿐
   - 메모리 관리는 OS가 담당

## 정확한 정답

### CPU (Central Processing Unit - 중앙 처리 장치)

**역할:**
- 컴퓨터의 **"두뇌"** 역할
- 모든 **연산(계산)**과 **명령어 실행**을 담당
- 프로그램의 명령어를 해석하고 실행

**주요 기능:**
1. **연산**: 덧셈, 뺄셈, 곱셈, 나눗셈 등 수학적 계산
2. **제어**: 다른 하드웨어 부품들에게 명령 전달
3. **명령어 실행**: 프로그램 코드를 한 줄씩 읽고 실행

**비유:**
- 사람으로 치면 **"두뇌"**
- 회사로 치면 **"CEO"** (일을 처리하고 결정하는 사람)

### RAM (Random Access Memory - 임시 기억 장치)

**역할:**
- 프로그램 실행에 필요한 **데이터를 임시로 저장**
- CPU가 빠르게 접근할 수 있는 **작업 공간**
- 전원이 꺼지면 데이터가 사라짐 (휘발성)

**주요 기능:**
1. **프로그램 코드 저장**: 실행 중인 프로그램의 명령어
2. **데이터 저장**: 프로그램이 사용하는 변수, 객체 등
3. **빠른 접근**: CPU가 매우 빠르게 데이터를 읽고 쓸 수 있음

**비유:**
- 사람으로 치면 **"책상"** (작업하는 공간)
- 회사로 치면 **"작업 테이블"** (자료를 펼쳐놓고 일하는 곳)

### CPU와 RAM의 협력 과정

프로그램이 실행될 때의 과정:

```
1. 프로그램 시작
   └─> 하드디스크(SSD)에서 프로그램을 RAM으로 로드

2. CPU가 RAM에서 명령어 읽기
   └─> RAM: "다음 명령어는 3 + 5를 계산하세요"

3. CPU가 연산 수행
   └─> CPU: "3 + 5 = 8 계산 완료!"

4. 결과를 RAM에 저장
   └─> RAM: "결과값 8을 저장했습니다"

5. 다음 명령어로 진행
   └─> 2번으로 돌아가서 반복
```

**실제 예시: 웹 브라우저 실행**

```
사용자: Chrome 브라우저 실행 버튼 클릭

1. OS: 하드디스크에서 Chrome 프로그램을 찾음
2. OS: Chrome을 RAM에 로드 (약 500MB)
3. CPU: RAM에서 Chrome의 첫 번째 명령어 읽기
4. CPU: "창을 그려라" 명령 실행
5. CPU: "메뉴바를 그려라" 명령 실행
6. CPU: "사용자 입력 대기" 명령 실행
7. RAM: 브라우저의 상태, 열린 탭 정보 등 저장

사용자: 유튜브 접속

8. CPU: 네트워크로 유튜브 데이터 요청
9. RAM: 받아온 영상 데이터 임시 저장
10. CPU: 영상을 화면에 표시
11. RAM: 재생 중인 영상 버퍼 저장
```

## 핵심 차이점

| 항목 | CPU | RAM |
|------|-----|-----|
| 역할 | **일하는 사람** (연산/실행) | **작업 공간** (저장) |
| 속도 | 매우 빠름 (GHz 단위) | 빠름 (ns 단위) |
| 용량 | 없음 (연산만 함) | 4GB ~ 32GB |
| 데이터 보존 | 해당 없음 | 전원 OFF 시 사라짐 |
| 비유 | 두뇌, 요리사 | 책상, 도마 |

## 추가 개념: 저장 장치와의 차이

### 3가지 주요 부품 비교

```
속도 빠름 ↑                    용량 작음 ↓
┌─────────────────────────────────────────┐
│ CPU (연산)                               │
│ - 역할: 계산하고 실행                    │
│ - 속도: 매우 빠름                        │
│ - 용량: 없음                             │
├─────────────────────────────────────────┤
│ RAM (임시 저장)                          │
│ - 역할: 실행 중인 데이터 보관            │
│ - 속도: 빠름                             │
│ - 용량: 8GB ~ 32GB                       │
│ - 휘발성: 전원 OFF 시 삭제               │
├─────────────────────────────────────────┤
│ 하드디스크/SSD (영구 저장)               │
│ - 역할: 파일, 프로그램 영구 보관         │
│ - 속도: 느림                             │
│ - 용량: 256GB ~ 2TB                      │
│ - 비휘발성: 전원 OFF 후에도 유지         │
└─────────────────────────────────────────┘
속도 느림 ↓                    용량 큼 ↑
```

### 왜 RAM이 필요한가?

**하드디스크에서 바로 실행하면 안 될까?**

```
하드디스크 속도: 100MB/s
RAM 속도: 25,000MB/s

→ RAM이 250배 빠름!
```

CPU는 매우 빠르기 때문에, 느린 하드디스크에서 직접 데이터를 읽으면 CPU가 기다려야 합니다.
RAM은 빠른 속도로 CPU에 데이터를 공급해줄 수 있습니다.

## 실생활 비유

### 요리사 비유

```
요리사 (CPU)
- 역할: 요리를 만듦 (연산 수행)
- 빠르게 일함

도마/작업대 (RAM)
- 역할: 재료를 올려놓는 공간
- 요리사가 빠르게 접근 가능
- 크기 제한 있음 (8GB = 작은 도마)
- 요리 끝나면 정리 (전원 OFF)

냉장고/창고 (하드디스크)
- 역할: 재료를 영구 보관
- 꺼내는데 시간 걸림
- 큰 용량
- 계속 보관됨
```

### 회사 업무 비유

```
직원 (CPU)
- 업무 처리 (문서 작성, 계산)

책상 (RAM)
- 지금 작업 중인 서류를 펼쳐놓음
- 빠르게 접근
- 퇴근하면 정리 (휘발성)

파일 캐비닛 (하드디스크)
- 완료된 문서 보관
- 찾는데 시간 걸림
- 영구 보관
```

## 프론트엔드 개발자가 알아야 할 이유

### 1. 메모리 관리

```javascript
// 나쁜 예: 메모리 누수
let bigData = [];
function addData() {
    bigData.push(new Array(1000000)); // RAM 계속 증가
}

// 좋은 예: 메모리 정리
function processData() {
    let tempData = new Array(1000000);
    // 작업 수행
    tempData = null; // 메모리 해제
}
```

### 2. 성능 최적화

```javascript
// CPU 집약적 작업 - CPU 부하
function calculatePrimes(max) {
    // 복잡한 계산 → CPU 사용
}

// 메모리 집약적 작업 - RAM 부하
const bigArray = new Array(10000000); // RAM 사용
```

### 3. 웹 애플리케이션 성능

- **이미지 로딩**: 하드디스크 → RAM → CPU → 화면
- **JavaScript 실행**: RAM에서 코드 읽기 → CPU 실행
- **상태 관리**: React state는 RAM에 저장

## 퀴즈

**Q1:** 프로그램을 많이 실행하면 컴퓨터가 느려지는 이유는?
<details>
<summary>답 보기</summary>
RAM이 부족해지면 하드디스크를 가상 메모리로 사용하는데, 하드디스크가 RAM보다 훨씬 느리기 때문입니다.
</details>

**Q2:** 컴퓨터를 껐다 켜면 실행 중이던 프로그램이 사라지는 이유는?
<details>
<summary>답 보기</summary>
RAM은 휘발성 메모리라서 전원이 꺼지면 저장된 데이터가 모두 사라지기 때문입니다.
</details>

**Q3:** CPU가 빠른데 왜 프로그램이 느릴까요?
<details>
<summary>답 보기</summary>
CPU가 아무리 빨라도 RAM이나 하드디스크에서 데이터를 읽어오는 속도가 느리면 CPU가 기다려야 하기 때문입니다. (병목 현상)
</details>
