## 질문

데이터베이스의 **인덱스(Index)**에 대해 설명해주세요.

다음 내용을 포함해서 답변해주세요:
1. 인덱스란 무엇인가?
2. 인덱스의 장점과 단점은?
3. 언제 인덱스를 사용해야 하나?

### 힌트
- 책의 목차나 색인을 생각해보세요
- 조회 성능과 삽입/수정/삭제 성능의 트레이드오프가 있습니다
- B-Tree, Hash 등 다양한 인덱스 구조가 있습니다

### 참고
인덱스는 데이터베이스 성능 최적화의 핵심 개념입니다. 프론트엔드 개발자도 백엔드 API 성능을 이해하기 위해 알아두면 좋습니다!
